import { FieldGroup, TextField, RadioGroup, Checkbox, FieldWrap } from '@deque/cauldron-react'

```js
import { FieldGroup } from '@deque/cauldron-react'
```

The FieldGroup component is designed to create accessible groupings of related form controls with proper labeling and semantic structure. You should use `FieldGroup` when:

- You need to group related form inputs that share a common purpose (like a set of checkboxes or radio buttons)
- You need to apply a single label to multiple related form controls
- You want to provide descriptive text that applies to an entire group of inputs
- You need to display validation errors that apply to the entire group rather than individual inputs

<Note>
  `FieldGroup` automatically appends `-label` to any id specified for the `FieldGroup` to the displayed label. If you need to reference the label id for any reason, you can reference `${id}-label`. If you prefer to have your own custom label, you can specify your own id by passing in a wrapping element to the `label` prop, e.g. `label={<span id="my-id">label</span>}`
</Note>


## Examples

### Grouping Checkboxes

The `FieldGroup` component is typically used in conjunction with [`FieldWrap`](./FieldWrap), which provides consistent styling and layout for form fields. To use `FieldGroup`, you can either nest them, or use the `as` prop on `FieldWrap` as shown in the example below.

```jsx example
<FieldWrap as={FieldGroup} label="Favorite way to prepare water">
  <Checkbox 
    id="grilled-water"
    label="Grilled" 
    name="prepare-water" 
    value="grilled-water"
    checked
  />
  <Checkbox 
    id="toasted-water"
    label="Toasted" 
    name="prepare-water" 
    value="toasted-water"
  />
  <Checkbox 
    id="steamed-water"
    label="Steamed" 
    name="prepare-water" 
    value="steamed-water"
  />
  <Checkbox 
    id="fried-water"
    label="Fried" 
    name="prepare-water" 
    value="fried-water"
  />
</FieldWrap>
```

### Grouping Radio Buttons

`RadioGroup` components should always be wrapped in a `FieldGroup` to ensure the set of radio controls has an associated group label.

```jsx example
<FieldWrap id="size" as={FieldGroup} label="How big is this thing?">
  <RadioGroup
    name="size"
    radios={[
      {
        id: '1',
        label: '1',
        value: '1'
      },
      {
        id: '2',
        label: '2',
        value: '2'
      },
      {
        id: '3',
        label: '3',
        value: '3'
      }
    ]}
  />
</FieldWrap>
```

### With Description

Descriptions provide additional context or instructions for the entire group of inputs. Use them to clarify requirements or explain how the grouped inputs relate to each other.

```jsx example
<FieldWrap
  as={FieldGroup} 
  label="Checkboxes" 
  description="Please check at least one checkbox"
>
  <Checkbox 
    id="yes"
    label="Yes" 
    name="checkboxes" 
  />
  <Checkbox 
    id="no"
    label="No" 
    name="checkboxes" 
  />
  <Checkbox 
    id="why"
    label="Why?" 
    name="checkboxes" 
  />
</FieldWrap>
```

### With Error Message

When validation fails for a group of form controls, you can display an error message that applies to the entire group. This is especially useful for groups of checkboxes where at least one selection may be required, or when certain combinations of selections are invalid.

```jsx example
<FieldWrap
  as={FieldGroup} 
  label="Please choose your favorite colors" 
  error="You cannot choose blue as one of your favorite colors"
>
  <Checkbox 
    id="red"
    label="Red" 
    name="colors" 
  />
  <Checkbox 
    id="blue"
    label="Blue" 
    name="colors"
    checked
  />
  <Checkbox 
    id="green"
    label="Green" 
    name="colors"
  />
</FieldWrap>
```
