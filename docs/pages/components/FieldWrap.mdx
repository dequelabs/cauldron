```js
import { FieldWrap } from '@deque/cauldron-react'
```

The FieldWrap component provides a background `Panel` appearance that is designed to ensure that Cauldron's
input components meet color contrast requirements (in all input states).

Input components (such as [Checkbox](./Checkbox) and [RadioGroup](./RadioGroup)) _should_ be wrapped in a
`FieldWrap`. Cauldron's input components have been designed specifically to meet color contrast requirements
against a `FieldWrap` background.

<Note>
  If you use a Cauldron input components without a `FieldWrap` (for example, within a `Modal`, where
  `FieldWrap`'s `Panel` appearance might not be appropriate), you _must_ verify that it meets accessibility
  requirements in all component states, especially for color contrast.
</Note>

## Examples

### Single Field

```jsx example
<FieldWrap>
  <TextField label="First name" />
</FieldWrap>
```

### Multiple Fields

```jsx example
<FieldWrap as="fieldset">
  <legend>
    Send us a message
  </legend>
  <TextField label="Email" />
  <TextField
    label="Message"
    multiline
  />
  <Button>
    Send Message
  </Button>
</FieldWrap>
```

### With FieldGroup

`FieldWrap` is often used together with [FieldGroup](./FieldGroup); `FieldWrap` ensures
an accessible appearance, and `FieldGroup` provides accessible structure and naming.

```jsx example
<FieldWrap id="size" as={FieldGroup} label="How big is this thing?">
  <RadioGroup
    name="size"
    radios={[
      {
        id: '1',
        label: '1',
        value: '1'
      },
      {
        id: '2',
        label: '2',
        value: '2'
      },
      {
        id: '3',
        label: '3',
        value: '3'
      }
    ]}
  />
</FieldWrap>
```
